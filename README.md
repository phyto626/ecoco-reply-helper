# ECOCO 社群回覆統整平台

ECOCO 客服團隊專用的社群回覆工具，讓客服人員能在幾秒內找到正確的標準化回覆並一鍵複製。

---

## 📁 檔案結構

```
repo 根目錄/
├── index.html          ← 主程式（單頁應用）
├── data/
│   ├── replies.json    ← 回覆範本資料庫
│   └── messages.json  ← 模擬留言資料（v1.0 未使用）
└── README.md
```

---

## 🚀 部署到 GitHub Pages

### 第一次部署

1. 在 GitHub 建立新 repo（建議命名：`ecoco-reply-helper`）
2. 將以下檔案上傳至 repo 根目錄，並確認 `data/` 資料夾結構正確：
   ```
   index.html
   data/replies.json
   data/messages.json
   README.md
   ```
3. 前往 repo 頁面 → **Settings** → **Pages**
4. Source 選擇 `Deploy from a branch`
5. Branch 選 `main`，資料夾選 `/ (root)`，點 **Save**
6. 約 1～2 分鐘後，網址會顯示在 Pages 設定頁面：
   ```
   https://{你的帳號}.github.io/ecoco-reply-helper/
   ```

### 更新內容

後續更新只需要在 GitHub 上直接編輯對應檔案，存檔後約 1 分鐘自動重新部署。

## 💻 本機測試

> ⚠️ 直接用瀏覽器開啟 `index.html` 會因瀏覽器安全限制（CORS）導致資料無法載入。

請使用以下其中一種方式：

**方法一：Python（推薦，不需安裝額外套件）**
```bash
# 在專案根目錄執行
python3 -m http.server 8080
```
開啟瀏覽器前往 `http://localhost:8080`

**方法二：VS Code Live Server**
安裝 VS Code 擴充套件「Live Server」，對 `index.html` 按右鍵 → Open with Live Server

## ✏️ 更新回覆內容

所有回覆範本集中在 `data/replies.json`，直接編輯此檔案即可。

### JSON 結構說明

```json
[
  {
    "id": "cat-01",
    "name": "💬 聊天互動 / 正面回饋",
    "description": "感謝留言、正面互動",
    "replies": [
      {
        "id": "reply-01-001",
        "title": "感謝支持（通用）",
        "body": "回覆內文\n第二行（用 \\n 換行）"
      }
    ]
  }
]
```

### 欄位說明

| 欄位 | 說明 |
|------|------|
| `id`（類別） | 格式 `cat-XX`，請勿更動 |
| `name` | 類別名稱，開頭含 emoji |
| `description` | 類別副說明 |
| `replies[].id` | 格式 `reply-XX-YYY`，每則不重複 |
| `replies[].title` | 情境標題，15 字以內 |
| `replies[].body` | 完整回覆內文，換行用 `\n`，可含 emoji |

### 新增一則回覆

在對應類別的 `replies` 陣列中加入新物件：

```json
{
  "id": "reply-01-004",
  "title": "新的情境標題",
  "body": "回覆內文第一行\n第二行\n第三行 💚"
}
```

> 💡 **ID 命名規則**：`reply-{類別編號}-{流水號}`，類別編號對應類別 `id` 的數字，流水號從 001 開始依序往後加。

## 📋 功能說明

| 功能 | 說明 |
|------|------|
| 類別選單 | 進入平台直接看到 10 個問題分類 |
| 回覆列表 | 點選類別後顯示對應的標準化回覆卡片 |
| 一鍵複製 | 點擊「複製此回覆」，2 秒確認後自動恢復 |
| 關鍵字搜尋 | 即時搜尋類別名稱、說明與回覆內文 |
| 搜尋結果 | 兩層結構：先顯示符合類別，展開看回覆 |
| 返回導航 | 回覆頁上方「返回類別」按鈕 |
