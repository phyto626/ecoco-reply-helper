# ECOCO 社群回覆統整平台 (v1.1)

ECOCO 客服團隊專用的社群回覆工具，讓客服人員能在幾秒內找到正確的標準化回覆並一鍵複製。最新版本已經整合了 Google Sheets CMS 後台、機台狀態查詢以及 AI 留言分類助手，大幅提升客服處理效率。

---

## 🚀 核心功能特色

### 1. 💬 回覆範本 (10大問題分類)
提供 10 大常見問題類別（如：滿倉換袋、機台維修、使用操作、APP 帳號問題等），並內建即時關鍵字搜尋，一鍵複製標準回覆，協助客服快速回應。

### 2. 🤖 AI 留言分類助手
- **自動分類**：客服人員只需貼上用戶的客訴或留言，AI 會自動分析語意並推薦最符合的問題類別。
- **推薦回覆與草稿生成**：AI 除了列出推薦的範本外，針對特殊問題也支援自動生成專屬回覆草稿，確認後可一鍵複製。

### 3. 📊 機台狀態查詢
- 點擊側邊欄「機台狀態」會另開新分頁，前往 ECOCO 機台狀態查詢系統（需登入帳號）。
- 查詢網址：`https://100.ecocogroup.com/stores`

### 4. ⚙️ 管理後台 (Google Sheets CMS 串接)
- 平台已完整串接 Google Sheets API 作為雲端資料庫。
- **即時管理**：客服主管可直接在平台的「管理後台」新增、編輯、刪除回覆範本或置頂最新系統公告。
- **無縫同步**：所有的操作皆會透過 API 即時同步寫入 Google Sheets，永久儲存。

## 🌐 部署方式 (GitHub Pages)

### 第一次部署

1. 在 GitHub 建立新 repo。
2. 將所有專案檔案上傳至 repo 根目錄。
3. 前往專案頁面 → **Settings** → **Pages**。
4. **Source** 選擇 `Deploy from a branch`。
5. **Branch** 選擇 `main`，資料夾選擇 `/ (root)`，點擊 **Save**。
6. 等候 1～2 分鐘，網址即可順利啟用，格式如：
   `https://{你的帳號}.github.io/{repo名稱}/`

### 本機端測試

> ⚠️ 注意：若直接雙擊點開 `index.html` 將會受限於瀏覽器的 CORS 跨網域存取限制，導致 API 無法正常抓取資料。

請使用下列其中一種方式開啟：
- **Python HTTP 伺服器**：在資料夾內執行 `python3 -m http.server 8080`，前往 `http://localhost:8080`。
- **VS Code Live Server**：安裝 Live Server 套件後，右鍵點擊 `index.html` 選取「Open with Live Server」。
